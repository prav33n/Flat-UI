10年前の起業以来、Glenbrookは弊社専用プラットフォームに基づいた先進的なデータサービスを顧客に提供することに焦点をおいてきました。GlenbrookのプラットフォームはDeep Web をナビゲートする能力基盤を構成し、ウェブページを分析、ウェブページのコンテキストを抽出し、これらのコンテキストに存在する非構造化情報をGlenbrook顧客のBig Data戦略構想に入力可能な構造化された事実へ変換します。

この方法によりGlenbrookは、金融サービス、情報技術、人事、モバイル、リスクマネジメント、出版、ソーシャルメディアにおける顧客に対するサービスの提供に成功してきました。

    Glenbrook独自の技術は6つの特許を取得しており、その他特許出願中の技術もいくつか有しています。プラットフォームは多言語とマルチバイト文字に対応可能です。

  • Deep Web vs. Surface Web
    • Surface Web は検索エンジン（例：Google）によって検索可能な統計ページで構成されます。サイズにすると500億ページを超えます。
    • それに比べDeep Webはより膨大で（1兆ページを上回る）、動的なウェブの部分です。これらのページはユーザーによるクエリーに応答して作成されます（例：“7月20日のシカゴからニューヨークまでの航空券代金はいくらですか？”）。動向は、増加し続ける情報がDeep Webに収納されるためです。
    • Glenbrookのプラットフォームは、いつDeep Web へ動くべきか（そしていつしないべきか）、指定のページについてのクエリーをどのように構造するか、その後、必要な情報をどのように効果的に見つけ抽出するかを理解しています。Surface Webに対してDeep Webが膨大なサイズで複雑であるため、これは必要不可欠です。

ビジネス記録の多言語変換

グローバル企業は日々、現在と将来の顧客記録を維持することに問題を抱えています。これらの記録は第三者（D&B）から得るかセールスとマーケティング組織を所有する企業から生成されているかのどちらかです。これらの記録を利用する際には、しばしば異なる言語であることや企業内での和解が容易でないことなどの課題があります。言語が異なるだけでなく、異なる言語における顧客データの記述方法の協定の違いもあります。例えば、中国語での住所要素の順序は英語を話す国々で使用されているそれとは反対です。

Glenbrookは問題を緩和する助けとなる特別な方法を開発しました。この方法は記録分析、複合翻訳技術、郵便協定に基づいた事実認証と対象国でのインターネット上の企業情報を組み合わせています。

世界規模のビジネスデータ収集

ビジネスを行うには、常に現在と将来の顧客に関するもっとも正確で最新の情報が必要不可欠です。インターネットはこれらのもっとも重要な情報源の一つです。企業のウェブサイトは素晴らしい情報源です。しかし、ビジネス情報のたったの2%が各企業のウェブサイト上に掲載されており、残りの98%の入手可能な情報はインターネット上の別のどこかから得らるのです（プレスリリース、記事、ブログなど）。その上、興味対象のウェブページのごくわずかのみがSurface Webにあり（検索エンジンでアクセス可能なウェブ部分）、残りの部分はDeep Webにあります。さらに困難なことに、検索された情報は構造化形式でウェブページに表示されることは滅多にありません（多くの場合、フリーフォームテキスト）。Glenbrookのプラットフォームはビジネスに関する事実情報の収集に最適であり、Deep Webをナビゲートし、ウェブページを分析、そこから独立コンテキストを抽出し、半構造化と非構造化情報をデータベースに保管可能な構造化事実へ変換することが可能です。

POI

モバイルの位置情報ベースのアプリケーションの増加に伴い、より多くの企業がユーザーに地元の企業や魅力に関する詳細情報と総合案内を提供することに注目しています。インターネットはこれらの興味対象（POI）の豊かな情報源です。Glenbrookはインターネットを通じて収集した情報を集約する能力を活用することによりPOI情報を収集し再読み込みします。
